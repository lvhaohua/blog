# 一、ECMA 规范阶段

Stage 0: strawman——最初想法的提交。
Stage 1: proposal（提案）——由TC39至少一名成员倡导的正式提案文件，该文件包括API事例。
Stage 2: draft（草案）——功能规范的初始版本，该版本包含功能规范的两个实验实现。
Stage 3: candidate（候选）——提案规范通过审查并从厂商那里收集反馈
Stage 4: finished（完成）——提案准备加入ECMAScript，但是到浏览器或者Nodejs中可能需要更长的时间。

# 1. 类 class

参考：https://es6.ruanyifeng.com/#docs/class

class Parent {
  constructor(name, age) {
    // 定义实例自身属性
    this.name = name;
    this.age = age;
    // Parent.prototype.like = ['basketball'];  // 泄漏了prototype
  }
  // color = ['red','blue']; // 会定义到实例属性，就像是 this.color = ['red','blue'];
  showName() {
    // 定义了parent.prototype.showName，会被所有子类共享，无法通过这种方式定义共享属性
    console.log(this.name);
  }
}

class Child extends Parent {
  constructor(name, age, fav) {
    // 若未置顶声明constructor，会默认生成
    super(name, age); // 声明了constructor就必须调用super，super是静态绑定[[HomeObject]].[[prototype]]，不是动态绑定
    this.fav = fav;
  }
  showAge() {
    console.log(this.age);
  }
}

var p1 = new Parent('p1', 50);
var c1 = new Child('c1', 20, ['basketball', 'football']);
c1.showName();
c1.showAge();

# 2. 模块化 module

var myModules = (function() {
  var allModules = {};
  function define(name, otherModuleName, fn) {
    var otherModule = [];
    for (var i = 0; i < otherModuleName.length; i++) {
      otherModule[i] = allModules[otherModuleName[i]];
    }
    allModules[name] = fn.apply(fn, otherModule);
  }
  function get(name) {
    return allModules[name];
  }
  return {
    define: define,
    get: get
  };
})();

myModules.define('a', [], function() {
  function logName(name) {
    console.log(name);
  }
  return {
    logName: logName
  };
});

myModules.define('b', ['a'], function() {
  function logAgeAndName(age, name) {
    a.logName(name);
    console.log(age);
  }
  return {
    logAgeAndName: logAgeAndName
  };
});

var a = myModules.get('a');
var b = myModules.get('b');
b.logAgeAndName(20, 'aa');

# 9. let const

相同点：

  • 块作用域外无法访问（var 只有全局作用域和函数作用域隔离）
  • 声明不会被提升，当执行流离开声明所在块作用域，该变量会立刻被销毁（var 会被提升）
  • 同一作用域中，不可重复声明（var 则可以）
  • 在全局声明中，不会被添加为全局对象的属性（var 则会）
var color = 'red';
console.log(window.color); // "red"  let animal = "panda";  console.log(window.animal); // undefined  console.log(animal in window); // false
1
2

不同点：

  • const 声明的是常量，必须进行初始化，let 则不是必须
  • 由 const 定义的变量，不可修改其引用，但可以修改该引用指向的值
const animal = 'panda';
animal = 'monkey'; // error   Uncaught TypeError: Assignment to constant variable.const fruits = {  like: "apple",  dislike: "orange"};fruit.like = "pear"; // that's ok

# 临时死区 TDZ（Temporal Dead Zone）

由 let 或 const 进行块级声明的变量不会像 var 被提升到作用域的顶部，所以在声明之前进行访问（包括 typeof）是会报错的：

if(true){  console.log(typeof color); // Uncaught ReferenceError: color is not defined  let color = "red";}
1

原因就是 JS 引擎在扫描代码时，会提前将由 let 或 const 声明的变量暂时放入 TDZ 中，直到执行过声明语句后，变量才会从 TDZ 中移除，所以在块级作用域中提前访问 let 或 const 声明的变量会导致报错，此外，而由 var 声明的变量则会提升至作用域顶部。

下面来看一个常见的示例： 期望依次打印 0-9

for (var i = 0; i < 10; i++) {
  setTimeout(function() {
    console.log(i);
  }, 100 * i);
} // 结果打印10个10
1
2
3
4
5

解决方案 1： IIFE 使用 IIFE 对每个变量 i 进行值传递给形参 num，从而保留每个变量 i 的值

for(var i = 0; i < 10; i++) {  (    function(num) {      setTimeout(function(){        console.log('num-->'num);        console.log('i-->',i); // 可见i仍全为10      }, 100 * i);    }  )(i);}// num依次打印0-9// i仍依次打印10
1

解决方案 2：使用块级作用域绑定 为每次循环创建一个新的作用域环境，所以每个作用域中变量 i 的值都是不同的

for (let i = 0; i < 10; i++) {
  setTimeout(function() {
    console.log('i-->', i);
  }, 100 * i);
}
