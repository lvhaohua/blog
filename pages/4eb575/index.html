DOM事件机制及事件委托

介绍了 DOM 的事件机制以及事件委托

# 一、 DOM 级别

# 1.1 DOM 0

element.onclick = function() {};
1
<input id='test' type='button' onclick='foo()' />;
// or
document.getElementByID('test').onclick = function() {};
// 注：无法为同一元素绑定多个相同事件

# 1.2 DOM 2

element.addEventListener(event - name, callback, useCapture);
1
  1. event-name: 事件名称，可以是标准的 DOM 事件
  2. callback: 回调函数，当事件触发时，函数会被注入一个参数为当前的事件对象 event
  3. useCapture: 默认是 false，代表事件将在冒泡阶段执行
document.getElementByID("test").addEventListener("click",funciton(){}); // 绑定
document.getElementByID("test").removeEventListener("click",funciton(){}); // 解除绑定
// IE9以下的IE浏览器不支持 addEventListener()和removeEventListener()
document.getElementByID("test").attachEvent('onclick',function(){}); // 绑定
document.getElementByID("test").detachEvent('onclick',function(){}); // 解除绑定

# 1.3 DOM 3

添加了更多的事件类型：

  • UI 事件，当用户与页面上的元素交互时触发，如：load、scroll
  • 焦点事件，当元素获得或失去焦点时触发，如：blur、focus
  • 鼠标事件，当用户通过鼠标在页面执行操作时触发如：dblclick、mouseup
  • 滚轮事件，当使用鼠标滚轮或类似设备时触发，如：mousewheel
  • 文本事件，当在文档中输入文本时触发，如：textInput
  • 键盘事件，当用户通过键盘在页面上执行操作时触发，如：keydown、keypress
  • 合成事件，当为 IME（输入法编辑器）输入字符时触发，如：compositionstart
  • 变动事件，当底层 DOM 结构发生变化时触发，如：DOMsubtreeModified
  • 同时 DOM3 级事件也允许使用者自定义一些事件

# 二、DOM 事件模型和事件流

在触发绑定事件后，将按以下规则传播:

  1. capturing phase 捕获阶段: 触发的事件会从 document 开始向下传播至目标元素，若中途有节点绑定了该事件则会执行该事件绑定的函数
  2. target phase 目标阶段: 事件到达目标元素,执行目标元素的绑定的事件处理函数
  3. bubbling phase 冒泡阶段: 触发的事件会从目标元素上升到 document 节点，若中途有节点绑定了该事件则会执行该事件绑定的函数

注：IE 事件模型没有“capturing phase”过程

<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
  </head>
  <body>
    <div id="a">
          <span id="b">bbbbb</span>
        </div>
      </div>
    </div>
    <script>
      let a = document.getElementById('a');
      let b = document.getElementById('b');
      a.addEventListener('click', function (event) {
        console.log('父节点a捕获');
      },true);
      a.addEventListener('click', function (event) {
        console.log('父节点a冒泡');
      });
      b.addEventListener('click', function (event) {
        console.log('子节点b捕获');
      },true);
      b.addEventListener('click', function (event) {
        console.log('子节点b冒泡');
      });
    </script>
  </body>
</html>

阻止冒泡

  1. event.stopPropagation 阻止捕获或者冒泡
  2. event.stopImmediatePropagation 阻止冒泡，且之后新绑定的同类型事件也阻止执行
  3. event.preventDefault 如果事件可取消，则取消该事件，而不停止事件的进一步传播

在这种情况下，如果对每个 li 都绑定一个 click 事件，开销是十分巨大的，所以我们用到了事件委托。

事件委托又叫事件代理，简单来说就是利用事件冒泡的特性，将子元素的事件响应统一交由父元素来管理。

// 利用事件冒泡的原理，利用上一层的节点元素进行事件代理
document.getElementByID('ul').addEventListener('click', function(event) {
  var target = event.target;
  if (target.tagName.toLowerCase() === 'li') {
    console.log(target.innerText);
  }
});
// IE event.srcElement

这里也引申出了事件对象 event，因为各个浏览器的事件对象不一样, 这里把主要的事件对象的属性和方法列出来

  • bubble ： 表明事件是否冒泡
  • cancelable ： 表明是否可以取消冒泡
  • currentTarget ： 当前时间程序正在处理的元素, 和 this 一样的
  • defaultPrevented： false ，如果调用了 preventDefualt 这个就为真了
  • detail： 与事件有关的信息(滚动事件等等)
  • eventPhase： 如果值为 1 表示处于捕获阶段， 值为 2 表示处于目标阶段，值为三表示在冒泡阶段
  • target || srcElement： 事件的目标
  • trusted： 为 ture 是浏览器生成的，为 false 是开发人员创建的（DOM3）
  • type ： 事件的类型
  • view ： 与元素关联的 window， 我们可能跨 iframe；
  • preventDefault()： 取消默认事件
  • stopPropagation()： 事件冒泡到父元素，阻止任何父事件处理程序被执行
  • stopImmediatePropagation()： 既能阻止事件向父元素冒泡，也能阻止元素同事件类型的其它监听器被触发**
  • IE 下的事件对象是在 window 下的，而标准应该作为一个参数, 传为函数第一个参数;

IE 的事件对象定义的属性跟标准的不同，如：

  • cancelBubble： 默认为 false, 如果为 true 就是取消事件冒泡;
  • returnValue； 默认是 true，如果为 false 就取消默认事件;
  • srcElement：这个指的是 target, Firefox 下的也是 srcElement;

# 4.1 如何实现事件先冒泡再捕获

参考链接：http://objcer.com/2017/08/14/Introduction-into-Events/

// addEventListener(type,listener,useCapture)
根据w3c标准，应先捕获再冒泡。若要实现先冒泡后捕获，给一个元素绑定两个addEventListener，其中一个第三个参数设置为false（即不捕获），另一个第三个参数设置为true（即捕获），调整它们的代码顺序，将设置为false的监听事件放在设置为true的监听事件前面即可。
