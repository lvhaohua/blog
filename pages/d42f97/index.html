前端实现文件下载压缩

伴随着前端的快速发展，前端能承担的任务也越来越繁杂，通常下载任务都是后端承担了大量的工作最终将下载地址返回到前端，本文将通过一个案例来分析前端如何替后端分担压力，从而提升下载效率

# 一、Overview

最近项目中有一个下载大量图片文件的需求，现有的下载整体完整流程如下（无缓存）：

  1. 前端发起下载请求
  2. 后端根据上传数据渲染图片并将渲染结果上传到存储中间件
  3. 后端从中间件下载图片后整体做压缩后再上传到存储中间件
  4. 后端将压缩包地址返回给前端
  5. 前端对压缩包进行下载

从流程中可以看出过程中的第 3 步与第 5 步是非常耗时且有重复和非必要逻辑的

在一个方案为50 页、单页尺寸1191x842、下载尺寸x4的情况下，最终的下载体积可达大约1G左右，从点击下载开始到前端正式开始下载耗时约2Min左右，紧接着才开始正式的下载过程，整体给到了用户非常不好的下载体验

所以为了减少整体下载耗时，考虑从以下 2 个方面来进行优化：

  1. 缩减页面渲染体积 -- 后端
  2. 压缩任务由前端来承担，避免后端多次与存储中间件之间的下载和上传任务 -- 前端

# 二、Todo & Not Todo

前后端共同优化后，同样的方案争取减少 50%以上的下载耗时，以现有 2Min 为例，优化后减少到 1Min 以内。

# 三、架构说明

基于现有的问题，重新对下载链路进行了新的设计优化

  1. 前端发起下载请求
  2. 后端渲染图片并将渲染结果上传到存储中间件后统一返回所有页面的图片地址集合
  3. 前端根据图片地址集合进行分组下载
  4. 全部下载完成后再前端进行打包压缩生成最终的压缩文件

# 四、详细设计

# 1. 分批下载

由于是前端进行压缩，所以后端返回地址集合后，前端需要将多个文件下载并存储在内存中

import axios from 'axios';

/**
 * 获取单个文件流
 *
 * @export
 * @param {string} url
 * @returns
 */
export function getSingleBlob(url: string) {
  return axios.get(url, {
    responseType: 'blob'
  });

/**
 * 获取多个文件流  ps: 是否需要考虑http2  http1.1两种情况？
 *
 * @export
 * @param {string[]} urls
 * @returns
 */
export function getMultipleBlob(urls: string[]) {
  return Promise.all(urls.map((url) => getSingleBlob(url)));
}

# 2. 生成 zip 文件

import JsZip from 'jszip';
import FileSaver from 'file-saver';
import { AxiosResponse } from 'axios';

/**
 * 导出zip文件
 *
 * @export
 * @param {AxiosResponse<any>[]} blobs
 * @param {string} fileName
 * @param {string} type
 */
export async function exportAll(blobs: AxiosResponse<any>[], type: string, fileName?: string) {
  try {
    // 创建zip
    const zip = JsZip();
    // 往zip中添加file
    blobs.forEach((blob, i) => {
      zip.file(`file-${i}.${type}`, blob);
    });
    // 生成最终的zip文件夹
    const zipFile = await zip.generateAsync({ type: 'blob' });
    // 命名下载
    FileSaver.saveAs(zipFile, fileName);
  } catch (error) {
    console.error(error);
  }
}

# 3. 封装成 npm 包

import JsZip from 'jszip';
import FileSaver from 'file-saver';
import axios, { AxiosResponse } from 'axios';

/**
 * 获取单个文件流
 *
 * @export
 * @param {string} url
 * @returns
 */
export function getSingleBlob(url: string) {
  return axios.get(url, {
    responseType: 'blob'
  });
}

/**
 * 下载所有文件流  ps: 是否需要考虑http2  http1.1两种情况？
 *
 * @export
 * @param {string[]} urls
 * @returns
 */
export function getMultipleBlob(urls: string[]) {
  return Promise.all(urls.map((url) => getSingleBlob(url)));
}

/**
 * 导出zip文件
 *
 * @export
 * @param {AxiosResponse<any>[]} blobs
 * @param {string} fileName
 * @param {string} type
 */
export async function exportAll(blobs: AxiosResponse<any>[], type: string, fileName?: string) {
  try {
    // 创建zip
    const zip = JsZip();
    // 往zip中添加file
    blobs.forEach((blob, i) => {
      zip.file(`file-${i}.${type}`, blob);
    });
    // 生成最终的zip文件夹
    const zipFile = await zip.generateAsync({ type: 'blob' });
    // 命名下载
    FileSaver.saveAs(zipFile, fileName);
  } catch (error) {
    console.error(error);
  }
}

/**
 * 下载zip包
 *
 * @export
 * @param {string[]} url 需要下载的文件url链接
 * @param {string} type 文件类型，如 'png'、'jpg'
 * @param {string} [fileName] 压缩包的名称，默认为 'download'
 * @returns
 */
export async function downloadZIP(url: string | string[], type: string, fileName?: string) {
  let blobs = null;

  if (typeof url === 'string' && url.length > 0) {
    blobs = await getSingleBlob(url);
  }

  if (Array.isArray(url) && url.length > 0) {
    blobs = await getMultipleBlob(url);
  }

  if (!blobs) {
    console.warn('download failed');
    return;
  }

  exportAll(blobs, type, fileName);
}
